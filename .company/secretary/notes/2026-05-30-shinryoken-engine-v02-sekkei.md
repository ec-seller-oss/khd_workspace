# 診療圏調査エンジン v02 設計書

> 決定（2026-05-29夜）：診療圏スキルを「福井のいつも作るファイル→スプシ→入力→福井の最終チェック→AI自動化」の量産体制にする。本書はその設計の確定版。

## 0. 結論（先に）

1. **福井の診療圏スプシは既に完成された精緻エンジン**（年齢別×男女別受療率 × 病名チェック × 昼夜補正）。**ゼロから作らない。これを土台に「入力収集」と「報告書化」をAI自動化する**のがv02。
2. 御代田v01の概算（受療率117/10万 一律）は、この精緻版に**置き換える**（下位互換だった）。
3. 福井さんの工数は最終的に**「病名選択・競合脅威度・事業計画の単価/経費前提」の最終チェックだけ**に圧縮 → 量産可能・単価/件数の天井が外れる。

## 1. 福井エンジンの解剖（つちはし整形 実物スプシより）

### 1-1. 入力ブロック（スプシのシート構成）
| ブロック | 主な入力項目 |
|---|---|
| ①基本情報 | 法人名・院名／所在地／最寄駅・距離方向／診療科目／診療所形態／開業区分（承継 or 新規）／承継・開業予定日／調査資料収集日／作成日／管轄保健所／担当者 |
| ②医師情報 | 院長氏名／生年月日／出身大学／専門科目／連絡先 |
| ③競合施設 | No./エリア/医療機関名/科目/住所/建物名 ＋ 詳細（院長名・年齢・出身校・開業年・診療時間・**推定1日外来数**） |
| ④診療圏 地区別人口 | 地区No./町丁名/**対象範囲(%)**／男女別・合計人口／世帯数／推定昼間人口／昼間人口比率 |
| ⑤齢別人口構成 | 5歳区分・10歳区分の人口・構成比・順位（高齢化率も） |
| ⑥**病名チェック** | 傷病分類リストに〇 → **診療科ごとに対象疾患を選択**（受療率合成の母体） |
| ⑦事業計画 主要数値 | 譲渡価格／建物帳簿価格／営業権／紹介手数料率／自己資金／借入金・期間・利率・返済方法／リース |
| ⑧患者数想定・単価 | 開院3ヶ月/2年目/3年目/4年目/5年目MAXの1日外来数／診療単価(千円/人)／その他収入／医療原価率／法定福利費率 |
| ⑨経費・概算事業費 | 水道光熱費・通信費・広告宣伝費・保険料・修繕費／駐車場敷金・開院費・火災保険料 等 |

### 1-2. 算出ロジック（核心＝v01より格段に精緻）
- **年齢別×男女別人口 × 年齢別×男女別受療率（確率小数）＝ 年齢別の推定患者数**
  - 例：女80歳以上の受療率＝0.0616、女45-49＝0.0046（年齢が上がるほど受療率が上がる）
- **昼夜補正**：診療所外来比率 **0.84** ＝ 診療所113.7 ÷（病院22.1＋診療所113.7）
  - 推定数 × 0.84 ÷（病床比2.1＋1）形の補正で「昼間/夜間」別の1日推定外来数を算出
- **昼間人口比率**（つちはし=0.68）を地区別人口に掛けて昼間人口を出す
- **病名チェック（〇）**で診療科の対象疾患だけ受療率を合成（整形は関節リウマチ/腰痛/骨折等に〇）
- 出力例（つちはし）：競合6施設の推定外来 計**約523人** / 自院1日推定外来数**120人**

### 1-3. 報告書アウトプット構成（スプシ内に章立てあり）
- Ⅰ調査概要（目的/承継計画/分析項目/期日）→ Ⅱ調査結果（総合結論/候補地/競合/人口/1日推定外来/ご注意）
- Ⅵ診療圏人口調査、Ⅶ1日推定外来数（算出表・年齢別受療率グラフ・昼夜別グラフ）
- 末尾に免責文（受療率表・住基・国勢に基づく／現地調査は別途、の定型）

## 2. AI自動化 vs 福井 最終チェックの切り分け（量産の肝）

| 工程 | 担当 | 中身 |
|---|---|---|
| 診療圏の設定（範囲・地区） | **福井 判断** | 何駅圏・どの町丁を入れるか（プロの土地勘） |
| 地区別・年齢別・男女別人口の収集 | **AI自動** | 住基・国勢から取得して④⑤へ投入 |
| 競合施設リスト収集 | **AI自動** | 名称・住所・診療時間・推定外来をWeb/地図から収集して③へ |
| 受療率テーブルの当て込み | **AI自動** | 患者調査の年齢別×男女別受療率を⑥の〇に応じて合成 |
| **病名（〇）の選択** | **福井 最終チェック** | 診療科の対象疾患の取り方（ここが結果を最も左右） |
| **競合脅威度の評価** | **福井 最終チェック** | ネット予約・評判・設備など数字に出ない要素 |
| 計算（昼夜補正・1日推定外来） | **AI自動** | 数式で自動 |
| **事業計画の単価・経費前提** | **福井 最終チェック** | 診療単価・患者数想定カーブ・経費水準 |
| 報告書ドラフト化（章立て・グラフ） | **AI自動** | つちはし型レイアウトに自動流し込み |

→ **福井の判断が要るのは「診療圏設定／病名選択／競合脅威度／事業計画前提」の4点だけ。残り約8割はAIが回す。**

## 3. 御代田 産婦人科への適用（v01→v02 差分）

- **病名チェックで〇にすべき（産婦人科）**：妊娠・分娩・産じょく／正常妊娠の管理（妊婦健診）／乳房及び女性生殖器の疾患（乳房は乳腺外科流出を一部減算）
- **必要データ**：上記傷病の**年齢別×男女別 外来受療率（女性中心）**を患者調査から取得 → v01の「117/10万 一律」を年齢別に分解
- **想定される変化**：女15-49に受療が集中する形に精緻化。御代田の女性人口構成（v01で女15-49≒2,911）と掛け合わせると、v01の「初期4〜10人/日」がより実態に寄った数字に補正される見込み
- 2次商圏（御代田＋軽井沢＋小諸南部 計約50,900／女15-49約8,600）はそのまま地区別人口④に投入可能

## 4. 量産フロー（確定）

1. 案件ごとに福井スプシをコピー
2. AIが人口（地区別・年齢別・男女別）と競合（名称・診療時間・推定外来）を収集して投入
3. AIが受療率を当て込み、昼夜補正で1日推定外来数を算出、報告書ドラフトを生成
4. **福井が4点（診療圏設定／病名選択／競合脅威度／事業計画前提）だけ最終チェック**
5. PDF出力 → 顧客提出

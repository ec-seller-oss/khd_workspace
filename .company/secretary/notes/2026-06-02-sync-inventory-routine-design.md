# SSoT→在庫表 転記ルーチン設計（確定・実装は次）2026-06

## 目的
物件管理マスターDB（SSoT）を唯一の真実源とし、軽量な「自社在庫表v2」へ
番号キーで転記（upsert）して二重管理を排す。

## 対象
- SSoT: 物件管理マスターDB `1XTPXFjtaoEKVlEaigP3U1VdYfG-IHa_9pqOiZ1-hA` / タブ「物」(gid=1649873874・57列)
- 転記先: 在庫表 `1ZOR5jn4UNj9pWPUfJeMVyGRH1YrzeKIThq8Lzw08nIU` / タブ「自社在庫表v2」(gid=489887089・30列)

## トリガー
手動。「在庫表同期」と言うと実行（自動監視はしない）。

## ロジック
1. 「物」タブA列(No=番号)をキーに全案件を読む
2. 在庫表v2のNo列と突合 → 既存番号は該当行を更新／無ければ末尾に追記（upsert）
3. 差分のみ書込。在庫表側の計算列（粗利(計算)・粗利率(計算)）は式のまま温存
4. スキーム列で「保有(インカム)」案件と「売買(再販)」を区別（在庫表は売買中心なので保有はフラグで併載）

## 列マッピング（物 → 在庫表v2）
| 在庫表v2 | ← 物タブ |
|---|---|
| No | No(番号) |
| 住所 | 所在地 |
| 最寄駅 / 徒歩分 | 最寄り駅 / 徒歩(分) |
| 用途地域 | 用途地域 |
| 土地㎡ / 床㎡ | 敷地面積㎡ / 占有面積㎡ |
| 仕入 | 仕入金額(→万円換算) |
| 売却 | 売上予定額 or 売却決済額(→万円) |
| バ粗利益 / バ粗利率 | 粗利益 / 粗利❹ |
| 調達銀行 / 調達額 / 自己資金 | 同名 |
| スキーム | スキーム |
| 売買契約/買決済/売却契約/売決済 | 購入契約/購入決済/売却契約/売却決済 |
| 担当 | ルート or 担当 |
| 売却先 | 買主 |
| 制約フラグ | 再建要件/弱み から要約 |
| 源 / 証憑 | "マイソク" / 概要書リンク |

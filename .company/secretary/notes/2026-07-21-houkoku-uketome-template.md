# 報連相テンプレ（鵜呑み防止・4行）

## なぜ作ったか
2026-07-21時点で止まっている案件を並べると共通点がある：

- 曾我/長西/内山（医療テナント）＝「福井の返信待ち」で数週間足踏み
- 細井さん（テナント）＝相手のオンライン秘書の「工事が終わるまで募集しない方がいい」を鵜呑みにして受け身化
- 高松（転売）＝「解体見積待ち」のまま自分から動いていない

**相手の発言を"最終回答"として受け止め、自分の次の一手をセットしないまま待ちに入る**のが停滞パターン。8大ブロッカー・黙る力5問・提案スピード＆クロージングレーダーは「自分が喋りすぎる/提案しすぎる」場面のブレーキとして機能しているが、**相手の発言を受け取った直後**をチェックする仕組みが抜けていた。ここを埋める。

## テンプレ（4行・その場で埋める）

```
【事実】○○さんが「____」と言った（要約・忖度せず、そのまま引用）
【判定】これは相手の最終回答か？ or 様子見・先延ばしか？（迷ったら様子見扱い＝鵜呑み厳禁）
【自分の一手】相手の返事待ちだけにしない。今日〜明日中に自分から動く一手を1つ
【期限】待つ場合も自分で区切る（日付を明記。"そのうち""追って"禁止）→ 過ぎたら自動で再アプローチ
```

## 運用ルール
1. 菊池さんが「〇〇さんと話した」「〇〇からこう言われた」と報告してきたら、秘書はこの4行を**その場で一緒に埋める**（後回し・夜まとめ禁止＝9原則④と同じ思想）。
2. 【判定】が「様子見・先延ばし」になったら、【自分の一手】と【期限】は必須（空欄で終わらせない）。
3. 様子見を示す典型ワード（検討します／様子を見て／工事・手続きが終わってから／また連絡します／一旦保留）が事実欄に出たら、秘書側が自動で赤旗を立てて判定を促す。
4. 【自分の一手】【期限】は作業DB（打席エンジン）にそのままタスク化する。相手待ちの案件を「待ち」のまま宙に浮かせない。
5. 期限を過ぎても相手から音沙汰がなければ、朝ブリーフのカレンダー差分チェック（2026-07-13〜運用）と同じ発想で「これ、いつやります？」と自動で聞く。


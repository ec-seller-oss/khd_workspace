# 京橋 本番LINE 運用メモ（2026-06-23 追補）

## ③「特典を受け取る」根絶（再デプロイ済）
- 原因＝LINE Harnessのマーケ用デフォルトintroメッセージ（DEFAULT_FORM_LINK_FLEX）。トラックリンクにintro_templateが無いと"特典"flexが出る。
- 対応：①クリニック用イントロテンプレ＋トラックリンク作成（cfeae83a…）②`~/.line-harness/apps/worker/src/services/intro-message.ts` の DEFAULT_FORM_LINK_FLEX をクリニック文面（ブランド赤）に書換え→`vite build && wrangler deploy`で再デプロイ（Version 9681d6b3）。
- 新・友だち追加URL（これを共有）：https://kyobashi-clinic.ec-seller.workers.dev/r/cfeae83a-0847-4a3d-ab4d-ea95039c6626?form=f02515a0-2082-4fef-bc83-ee6f7aff670d

## ② フォーム送信→確認＋タグ
- 予約フォーム送信→「予約リクエストを受け付けました…」自動返信＋タグ「予約リクエスト」付与。
- 問診フォーム送信→お礼＋タグ「問診記入済」付与。
- ※日時入りの「予約確定」「前日リマインド」自動化は予約システム/スケジューラ連携が必要＝次フェーズ。

## ① クリニックに確認する固有値（先生ヒアリング用）→ 揃ったらClaudeが反映
- 診療時間（平日/土・午前午後の区切り・昼休み）
- 休診日（曜日・祝日・年末年始）
- 電話番号 / 住所・最寄り駅・駐車場
- マイナ保険証・本人確認の運用
- CPAP定期の案内可否・頻度
- 化学物質過敏症の対応方針（証明書の可否）
- 予約枠（初診/再診/CPAPの枠・予約可能曜日）
- 問診・予約回答の送信先・保管・閲覧者／個人情報の同意文言
- 公式アカウント名（テスト→本番の正式名）

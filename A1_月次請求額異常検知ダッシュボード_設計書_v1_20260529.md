## 1. 目的・問題定義

### 解決したい問題
2026 Q1で **JAL連携が3/09で切れ、5月末まで気づかなかった**。結果、抜け4件・二重170万を遡って修正する大手術になった。**月次で連携事故を早期発見できれば、四半期点検の負荷が激減**する。

### 数値目標
- 連携事故の発見ラグ：**最大3ヶ月（現状）→ 1ヶ月以内**
- 四半期点検の工数：**1〜2時間（現状Tier1）→ 30分**
- 還付取りこぼし：**ゼロ化**

## 2. アーキテクチャ

```
[Gmail]
  ├─ AmericanExpress（91005 / 41001 / 1005[MB]）
  ├─ 楽天カード（ブラックVisa）
  ├─ セゾン（プラチナビジネス・SAISON Digital）
  ├─ JCB（ANA ToMePASMO GOLD）
  └─ ビューカード（JAL）※要追加調査
     │ 月初に「請求金額確定メール」を自動抽出
     ▼
[Claude（月初トリガー）]
     │ メール本文から「請求金額・カード下4-5桁・振替日」を抽出
     ▼
[Notion DB 月次カード請求額]（新規）
     │
     │ ＋
     ▼
[MF仕訳帳]（菊池さんがエクスポート or 別ルートで取得）
     │ カード別未払金 期末残・月次計上額を抽出
     ▼
[Notion ビュー: 異常検知ダッシュボード]
     ├─ 月次推移グラフ
     ├─ カード別 請求額 vs MF計上 の差異
     └─ 🔴 差異あり → 早期警告（Claude→Gmail通知 or Notion strikethrough）
```

## 3. Notion DB設計（新規作成）

### DB「月次カード請求額」
| プロパティ | 型 | 例 |
|---|---|---|
| タイトル（YYYY-MM_カード） | Text | 2026-05_AmEx_1005 |
| カード名 | Select | AmEx_MB / 楽天Visa / JAL_Suica / 等 |
| 下4-5桁 | Text | 1005 |
| 年月 | Date（年月） | 2026-05 |
| 請求金額（カード会社発表） | Number | 447,232 |
| MF計上金額（同月引落予定） | Number | （手動 or 連携） |
| 差異 | Formula | abs(請求-MF計上) |
| 振替日 | Date | 2026-06-10 |
| メール受信日 | Date | 2026-05-24 |
| 異常フラグ | Checkbox | 差異>1,000円で自動チェック |
| Gmail ThreadID | Text | （リンク用） |

### ビュー
- 月次推移（カード別グラフ）
- 異常フラグONのみ（早期警告ボード）
- 直近3ヶ月（操作頻度高）

## 8. 不正請求検知（拡張機能 / 2026-05-29追加）

### 検知パターン4種
| # | パターン | 検知ロジック | データソース |
|---|---|---|---|
| ① | **見覚えのない加盟店** | 過去6ヶ月に未出現の加盟店からの請求を一覧化 | MF仕訳（過去6ヶ月） |
| ② | **異常な金額** | 過去平均の3倍以上の単発取引 | MF仕訳 |
| ③ | **短時間連続請求** | 同カードで30分以内に複数請求 | Gmail（利用通知メール） |
| ④ | **海外加盟店** | 日本以外のドメイン／外貨表記 | Gmail or MF |

### 実装レイヤー（ハイブリッド）
- **速報レイヤー（リアルタイム）**：AmExの「カードご利用金額のお知らせ」メールを毎日Gmail監視 → 30分連続・5万円超・海外 を即警告
- **月次精査レイヤー**：月初にMF仕訳の過去パターンと比較 → 新規加盟店リストを菊池さんに提示→「これは正常？」を1件ずつ確認

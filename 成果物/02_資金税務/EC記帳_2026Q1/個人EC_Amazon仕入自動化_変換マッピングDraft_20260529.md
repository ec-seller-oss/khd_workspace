## 結論：手入力162件の正体と、自動化の対象

2025準拠CSV（`KHD記帳_個人2026/個人EC仕訳_2026年2-4月_2025準拠.csv`）の建付け：

- **原資チャージ**＝`借方:獲得ポイント残高/対象外 ← 貸方:未払金/JALSuicaゴールド`（カード明細から計上。済）
- **実仕入**＝「Amazon注文明細から**別途**仕入高計上」← ★ここが162件の手入力地獄＝**今回自動化する本体**

つまり自動化対象は「注文1件 → 仕入高仕訳1行」の量産。原資チャージ側は既存ロジックのまま。

## ソース列（Amazonビジネス注文履歴＝クーパン2／全80列中の使用列）

| 用途 | ソース列名 |
|---|---|
| 取引日 | **注文日**（このレポートはJST表記。例 2025/07/28） |
| 仕入先補助の判定 | **アカウントグループ**＝`KIKUCHI HOLDINGS K.K.`→クーパン2 |
| 金額(税込) | **注文の合計（税込）** |
| 金額(税抜) | **注文の小計（税抜）** |
| 消費税 | **注文の消費税額** |
| 税率 | **商品の小計（税率）**＝10% |
| インボイス判定 | **適格請求書発行事業者登録番号**（`T...`有=適格／空=非適格） |
| インボイス発行者 | **適格請求書（または支払い明細書）発行者名** |
| 摘要 | **商品名** / **出品者名** |
| ID(メモ) | **注文番号**（249-...） / **ASIN** |
| 支払 | **支払い方法**＝`ギフトカード`（=チャージ済原資の取崩） |
| 配送/割引 | **注文の配送料および手数料（税抜）** / **注文の割引（税込）** |

## ターゲット仕訳（推定ルール／注文1件=1仕訳）

```
取引日       ← 注文日
借方勘定科目 ← 仕入高
借方補助科目 ← クーパン2
借方税区分   ← 課仕 10%（インボイスT有=適格／空=要区分判定）
借方金額     ← ★税込経理なら[注文の合計(税込)]／税抜経理なら[小計(税抜)]
貸方勘定科目 ← 獲得ポイント残高（支払=ギフトカード＝チャージ原資の取崩）
貸方税区分   ← 対象外
貸方金額     ← 同額
摘要         ← 商品名（or 出品者名）
メモ         ← 注文番号 / ASIN / インボイス番号
```

出力フォーマットは確立済みの**MFインポート19列**（`reference_tax_audit_methodology §9`）。UTF-8 BOM。

## ★橋本税理士に確認すべき税務論点（自動化を回す前提条件）

1. **税込経理 or 税抜経理**＝仕入高にどの金額列を入れるか（現状ルールの確認）
2. **貸方=獲得ポイント残高（資産取崩）で確定か**（2025準拠の建付けを踏襲してよいか）
3. **インボイス番号"無し"出品者の税区分**＝適格／非適格経過措置80%控除の扱い
4. **配送料・割引の扱い**＝仕入高に含めるか、荷造運賃/別科目に分けるか
5. **輸出免税売上との対応**＝仕入税額控除は全額か（個別/一括対応）

→ 1〜5が固まれば、変換スクリプト（注文履歴CSV→MF仕訳CSV）を実装してQ1の162件を一括生成できる。
